# Token 优化

在 AI 编程工具中高效使用 Token，在成本、效率、效果之间取得最优平衡的实践方法。

## Token 基础

- 1 Token ≈ 3-4 个英文字母 或 0.5-1 个中文字（模型不同有差别）。
- AI 每次响应都重新读取整个对话历史——对话越长，每次请求消耗越多。
- 成本：1000 次低效对话 vs 100 次高效对话，成本可相差 10 倍。

## 提问技巧

### 明确需求（减少返工）
- 指定角色（"假设你是 React 开发者"）
- 明确格式（"只给代码，不要解释"）
- 提供背景（项目技术栈、当前状态）

### 先确认再执行
```
[你的需求]
请在回答前先问我问题，一次只问一个，直到你有 95% 的信心理解我的需求，然后再给出方案。
```

## 工具使用技巧

| 技巧 | 说明 |
|------|------|
| Plan 模式 | 复杂任务先规划再执行，避免方向错误 |
| Debug 模式 | 加打印→跑测→分析日志，系统定位问题 |
| 精确引用 | 只引用相关代码片段，用 `@文件:函数` 精确到函数 |
| 快捷键编辑 | 简单修改用 Ctrl+K 内联编辑，不必开对话 |

## 模型分层策略

- **简单任务**（格式化、语法检查、简单增删改）：用便宜/快速模型
- **复杂任务**（架构设计、复杂算法、难以定位的 Bug）：用高级模型（Claude Opus 4.7）
- 日常开发可节省 60-80% Token 成本

对应 [[../entities/claude-code]] 的 `/model opusplan` 命令：规划用 Opus，执行切回 Sonnet。
（来源：[[../sources/2026-03-claude-code-hidden-commands]]）

## 对话管理

- 上下文使用量达到 60% 时开新对话。
- 一个会话只处理一个任务。
- 使用 `/btw` 在长任务中免污染插问（来源：[[../sources/2026-03-claude-code-hidden-commands]]）。
- 使用 [[agent-skills]] 按需加载专业知识，不一直占用上下文。
