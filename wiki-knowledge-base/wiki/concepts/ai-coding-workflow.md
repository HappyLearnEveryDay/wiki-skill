# AI 辅助编码工作流

使用 [[../entities/cursor]]、[[../entities/trae]] 等 AI 编程工具进行软件开发的最佳实践流程。

## 核心原则

1. **先明确需求，再动手**：想清楚画面感和细节，避免方向错误。（来源：[[../sources/2026-02-activity-viz-tool]]）
2. **给足信息再梭哈**：需求明确后给最详细的描述，用最好的模型一次生成。
3. **及早引入版本控制**：改动觉得 OK 就提交一次；若 AI 反复无法达到需求，回滚到稳定版本重新提问。

## 开发流程

```
明确需求（纸上/Ask 模式探讨）
    ↓
Plan 模式规划（复杂任务）
    ↓
Agent/Solo 模式实现
    ↓
审查代码 → 局部修改（Ctrl+K）
    ↓
测试验证（截图喂 AI 描述界面问题）
    ↓
Git 提交（稳定版本）
    ↓
迭代优化
```

## 与 AI 协作技巧

| 场景 | 技巧 |
|------|------|
| 界面问题难描述 | 截图粘贴到输入框直接发给 AI |
| AI 理解错误 | 明确指出问题并重新教育 AI |
| 大型改写前 | 让 AI 根据 git 暂存区内容反向总结需求并审核 |
| 局部修改 | `Ctrl+K` 在代码块内操作，避免改动过多 |
| 卡住了 | 不要反复鞭策，回滚到稳定版本重新提问 |

## 工具选择

- **[[../entities/cursor]]**：专业度高，功能完整，适合有一定编程基础的用户。
- **[[../entities/trae]]**：国内体验好，适合零基础入门。

## 不同复杂度的对应策略

- **简单修改**：`Ctrl+K` 内联编辑（不必开对话）
- **单一功能**：直接 Agent 模式，给详细需求
- **复杂需求**：Plan 模式先规划，确认后再执行
- **调试问题**：Debug 模式（加打印→跑测→分析日志→修复）
