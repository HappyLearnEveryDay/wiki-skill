# 活动可视化工具[Cursor]

**来源**：星辰技术专项组 | 2026年2月

## TL;DR

一个用 AI 辅助开发的游戏运营活动可视化工具的完整实战案例，技术栈为 Erlang + Python + Vue。记录了从需求分析、与 AI 共同探讨方案、逐步迭代（Erlang 后端 → Python 图形 → Vue 前端）到集成部署的全过程。

## 核心主张

### 开发流程原则
- 动手前先明确需求，想清楚画面感和细节，避免方向错误。
- 复杂任务先在 Ask 模式与 AI 商定方案，再动手实现。
- 需求明确后给 AI 尽可能详细的描述，用最好的模型"梭哈"一把。

### 迭代与版本管理
- 在工具开发中尽早引入 Git 版本控制；觉得改动 OK 时就提交一次。
- 若 AI 反复无法达到需求，不要继续鞭策，应回滚到稳定版本重新提问。

### 与 AI 协作技巧
- 遇到 AI 理解错误（如错误引用项目内部接口）：明确指出问题并重新教育 AI。
- 不好描述的界面问题：截图粘贴到输入框直接给 AI 看。
- 对大型改写：让 AI 根据 git 暂存区内容反向总结需求并审核可行性，快速 review。
- 用 `ctrl+k` 在代码块范围内做局部修改，避免改动过多地方。

### 技术路线演进
- Erlang 解析活动数据 → Python 绘图 → 发现 Python 脚本界面太差 → 改为 Vue 前端。
- 最终集成进已有项目："请帮我把 xxx 文件夹内的内容集成进 xxx 文件夹内"。
